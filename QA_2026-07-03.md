# Live QA — Party OS + DM dock (v0.1.67, 2026-07-03)

Consolidated from DESIGN §15 Rounds 5–17. Everything below was built + phone-verified
where possible; the **executor (GM) / TV** halves could not be driven from the phone
client, so they need a live pass. Order: reload → test one item → note pass/fail.

## 0. Setup (do first)
- [ ] **Hard-reload (Ctrl+Shift+R)** the GM/executor client, every phone, and the TV
      client. (Plain F5 keeps stale CSS — this session hit that repeatedly.)
- [ ] Confirm the 4-PC group exists with the right members; PCs assigned to users
      (User Config) for distinct player colours.

## 1. DM panel — dock (GM client)
- [ ] Right-edge tab rail shows **D20 (Request rolls)** and **🐾/👥 (Players)**;
      **▦ (Party order)** appears only when packed.
- [ ] Click a tab → same-height flyout opens on the right; the tab rail moves to the
      flyout's **right** edge. Click the tab again / the **✕** closes it.
- [ ] **Drag a header** (flyout header or a section header) → the whole dock moves;
      the **✕** and dropdowns still work (don't start a drag).
- [ ] Open a flyout near the screen's right/bottom edge → the panel shifts so the
      **whole flyout stays on screen** (clampPos fix).

## 2. Request rolls (GM → phones)
- [ ] D20 tab: three controls on one line — **Save/Check**, **ability**, **targets
      dropdown** (opens *out of the frame*; grouped members preselected).
- [ ] **Request rolls** → each selected player's phone shows a gold **"Roll DEX
      save"** card; tapping opens the native roll dialog (Dice-So-Nice animates).
- [ ] An **offline** owner's roll is auto-rolled by the executor (posts to chat).

## 3. Party order (pack → arrange → lock → travel → disperse)
- [ ] Cluster the 4 PCs → main panel shows **Form up** → click → tokens collapse to
      the group token; the **Party-order tab auto-opens**; phones jump to the order tab.
- [ ] Grid: **empty squares inert**, member cells **outlined in player colour**,
      **click A then B swaps** (no stacking).
- [ ] **↺ [Lock in] ↻** — Lock in disabled until every cell is unique; rotate turns
      the party **in the grid** (all 8 directions).
- [ ] After Lock in: phones get the **move pad**; the pad moves the **group token**;
      **TV follows** keeping ~25 ft around it. **Manual zoom holds; splitting zooms
      out; regrouping returns to the set zoom** (no 5-ft-move "orbital" jump).
- [ ] **Auto-facing:** walk the party each direction (incl. diagonals) → the grid
      re-orients; **Disperse** deploys facing travel.
- [ ] Park near a wall → **red blocked cells**; Disperse → *"Can't deploy while X is
      behind a wall…"* → arms **"Disperse anyway"** (off-map is the only hard stop).
- [ ] **Release/Combine:** select a member (travel) → **Release** → their token spawns
      beside the party, that player's phone jumps to the PC's Explore with the token
      selected; walk back within 1 sq → **Combine**.

## 4. Party OS (phone, packed)
- [ ] Tabs: **Party / Party order / Exploration / Inventory / Journal / My-sheet**.
- [ ] **Party view** roster (HP/HD/AC/speed/Perception-Investigation-Insight + class
      icons); **Group portrait** button → generator seeded from the party.
- [ ] **Exploration:** travel pad + **travel pace** (set the group's land/water/air on
      its sheet) + **Perception/Stealth/Survival** buttons (roll natively, on-screen dice).
- [ ] **Journal:** filter on top, posts scroll, composer pinned bottom.
- [ ] **My sheet:** full PC UI; tokenless rolls work (**group Stealth check** — verified).

## 5. Vision / rings / TV (TV + GM)
- [ ] `MobileCommand.fixPcTokens()` on the GM console → existing PC tokens get the
      **player-colour dynamic ring** (+ colour-over-subject) + tiny glow + darkvision
      saturation. New pack/disperse applies it automatically.
- [ ] Dark area on the TV: muted-colour world, **full-colour tokens**, **no orange
      controlled-token boxes**; player-colour rings legible.
- [ ] **TV compass** bottom-right (north arrow only).

## 6. Players tab + colours (GM → phone)
- [ ] Players tab: pick a player → their tokens in a **2-row horizontal grid, highest
      level first**; **drag a thumbnail onto the map** spawns a token; **double-click**
      opens the sheet (TV excluded).
- [ ] Tap the **palette** by a player → that player's phone shows a full-screen
      **"Pick your colour"** overlay; pick → their ring/banner/pips follow; **the
      colour persists next session** (native `user.color`).
